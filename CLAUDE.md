## Git & GitHub Workflow

After completing any meaningful unit of work, commit and push to GitHub. Don't batch up large amounts of unrelated changes — commit incrementally with clean, descriptive messages so work is never lost.

- Stage specific files rather than `git add -A`
- Write commit messages in the imperative mood ("Add feature", "Fix bug", "Update styles")
- Push to `main` after every commit: `git push origin main`
- Never skip hooks or force-push to `main`
